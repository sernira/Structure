فصل ۱: ساختار پایهٔ سند HTML

1.1 مستندات و تاریخچه

زبان HTML (HyperText Markup Language) یک زبان نشانه‌گذاری (Markup Language) است که برای ساختاردهی به محتوا در وب استفاده می‌شود. برخلاف زبان‌های برنامه‌نویسی (مانند پایتون یا جاوااسکریپت) HTML یک زبان برنامه‌نویسی نیست بلکه یک زبان توصیفی است که با استفاده از تگ‌ها (tags) و ویژگی‌ها (attributes) ساختار و معنای محتوا را مشخص می‌کند. HTML به مرورگرها دستور می‌دهد که چگونه محتوا (مانند متن، تصاویر، ویدئوها، لینک‌ها و...) را نمایش یا سازمان‌دهی کنند.

هدف اصلی HTML:

  • ساختاردهی به محتوا (مانند سرفصل‌ها، پاراگراف‌ها، لیست‌ها).
  • ایجاد پیوندهای بین صفحات وب (Hyperlinks).
  • فراهم کردن چارچوبی برای استایل‌دهی (با CSS) و تعاملات پویا (با جاوااسکریپت).
  • بهبود دسترسی‌پذیری و سئو با استفاده از تگ‌های معنایی.
نشانه‌گذاری (Markup): نشانه‌گذاری به استفاده از تگ‌ها و ویژگی‌ها برای توصیف ساختار و معنای محتوا در یک سند اشاره دارد. در HTML نشانه‌گذاری شامل تگ‌هایی است که محتوا را در بر می‌گیرند و به مرورگرها اطلاع می‌دهند که چگونه آن را تفسیر کنند. نشانه‌گذاری در HTML به دو نوع اصلی تقسیم می‌شود:
  • نشانه‌گذاری ساختاری (Structural Markup):
    • تگ‌هایی که ساختار سند را تعریف می‌کنند، مانند <p> برای پاراگراف‌ها، <ul> و <ol> برای لیست‌ها. مثال: <p> این یک پاراگراف است </p> به مرورگر می‌گوید که متن داخل تگ یک پاراگراف است.
  • نشانه‌گذاری معنایی (Semantic Markup):
    • تگ‌هایی که معنای محتوا را مشخص می‌کنند، مانند <footer> برای پاورقی. این تگ‌ها به موتورهای جستجو و فناوری‌های کمکی (مانند صفحه‌خوان‌ها) کمک می‌کنند تا محتوای صفحه را بهتر درک کنند. مثال: <article><p>example text</p></article>

دو نهاد کلیدی در توسعه استانداردهای وب WHATWG (Web Hypertext Application Technology Working Group) و W3C (World Wide Web Consortium) هستند که هرکدام نقش متفاوتی در تعریف مشخصات HTML ایفا می‌کنند.
  • WHATWG در سال 2004 توسط توسعه‌دهندگان مرورگرهای اصلی (مانند اپل، موزیلا و گوگل) تأسیس شد تا استانداردهای وب را به‌صورت پویا و متناسب با نیازهای واقعی توسعه‌دهندگان و مرورگرها به‌روزرسانی کند. استانداردی که WHATWG ارائه می‌دهد به نام Living Standard شناخته می‌شود به این معنا که یک سند زنده و در حال توسعه است که به‌جای انتشار نسخه‌های ثابت به‌صورت مداوم به‌روزرسانی می‌شود. این رویکرد امکان انعطاف‌پذیری و تطابق سریع با فناوری‌های جدید را فراهم می‌کند.
  • W3C یک سازمان بین‌المللی است که توسط تیم برنرز-لی تأسیس شد و بر توسعه استانداردهای وب با تمرکز بر دسترسی‌پذیری پایداری و توافق جهانی نظارت دارد. W3C نسخه‌های مشخصی از HTML (مانند HTML5) را منتشر می‌کند که به‌عنوان استانداردهای ثابت شناخته می‌شوند. این نسخه‌ها معمولاً پس از توافق‌های گسترده بین اعضای W3C (شامل شرکت‌ها، دانشگاه‌ها و سازمان‌ها) نهایی می‌شوند. W3C بر جنبه‌های نظری و استانداردسازی دقیق‌تر تأکید دارد و گاهی رویکرد محافظه‌کارانه‌تری نسبت به WHATWG دارد.

از زمان پیدایش HTML در اواخر دهه 1980 نسخه‌های مختلفی از آن منتشر شده است که هر یک ویژگی‌ها و بهبودهای خاصی را به همراه داشته‌اند. در ادامه نسخه‌های HTML به ترتیب زمانی بررسی می‌شوند:

1. HTML 1.0 (1989-1992)

نسخه HTML 1.0 توسط تیم برنرز-لی مخترع وب در CERN توسعه یافت. این نسخه به‌عنوان اولین نسخه رسمی HTML در سال 1990 معرفی شد.
  • بسیار ابتدایی و محدود، با حدود 18 تگ ساده (مانند <h1>، <p>، <a>، <ul>، <ol>).
  • تمرکز بر ایجاد اسناد متنی با قابلیت پیوند (hyperlink) بین صفحات.
  • بدون پشتیبانی از تصاویر، جداول، یا استایل‌دهی پیشرفته.
  • عدم سازگاری گسترده بین مرورگرها به دلیل نبود استاندارد مشخص.

2. HTML 2.0 (1995)

نسخه HTML 2.0 توسط IETF (Internet Engineering Task Force) به‌عنوان اولین استاندارد رسمی HTML منتشر شد.
  • معرفی تگ‌های جدید مانند <form> برای ایجاد فرم‌های تعاملی.
  • پشتیبانی از تصاویر با تگ <img>
  • محدود به محتوای متنی و تصاویر ساده.

3. HTML 3.2 (1997)

نسخه HTML 3.2 توسط W3C (World Wide Web Consortium) منتشر شد و به‌عنوان اولین نسخه تحت نظارت این سازمان شناخته می‌شود.
  • پشتیبانی از جداول با تگ <table> برای چیدمان بهتر محتوا.
  • معرفی تگ‌های جدید مانند <font> برای کنترل ظاهر متن.
  • سازگاری با مرورگرهای اولیه مانند Netscape و Internet Explorer.

4. HTML 4.0 و HTML 4.01 (1997-1999)

نسخه HTML 4.0 در سال 1997 و نسخه بهبودیافته HTML 4.01 در سال 1999 توسط W3C منتشر شد. این نسخه‌ها استانداردهای وب را به سطح جدیدی ارتقا دادند.
  • جداسازی ساختار از ظاهر با تشویق به استفاده از CSS (Cascading Style Sheets).
  • پشتیبانی از اسکریپت‌نویسی (مانند جاوااسکریپت) برای تعاملات پویا.
  • معرفی تگ‌های جدید مانند <object> برای جاسازی محتوای چندرسانه‌ای.
  • وابستگی به افزونه‌های خارجی (مانند ادوبی فلش) برای محتوای چندرسانه‌ای.

5. XHTML 1.0 و XHTML 1.1 (2000-2001)

نسخه XHTML (Extensible HyperText Markup Language) به‌عنوان نسخه‌ای مبتنی بر XML از HTML4 معرفی شد. XHTML 1.0 در سال 2000 و XHTML 1.1 در سال 2001 منتشر شدند.
  • قواعد سخت‌گیرانه‌تر (مانند بستن اجباری همه تگ‌ها، استفاده از حروف کوچک برای تگ‌ها و ویژگی‌ها).
  • سازگاری با استانداردهای XML، که امکان پردازش توسط ابزارهای XML را فراهم می‌کرد.
  • پذیرش محدود توسط مرورگرها و توسعه‌دهندگان به دلیل پیچیدگی.

6. XHTML 2.0 (لغو شده)

نسخه XHTML 2.0 پروژه‌ای بود که توسط W3C آغاز شد اما در سال 2009 به دلیل عدم پذیرش و رقابت با HTML5 لغو شد.
  • تمرکز بر جداسازی کامل ساختار و رفتار.
  • ناسازگاری با نسخه‌های قبلی HTML و XHTML.

7. HTML5, Living Standard (2014)

نسخه HTML5 ابتدا توسط WHATWG به‌عنوان یک استاندارد زنده (Living Standard) توسعه یافت و در سال 2014 توسط W3C به‌عنوان استاندارد رسمی منتشر شد.
  • تگ‌های جدید مانند <article> - <section> - <nav> - <header> - <footer> برای بهبود ساختار، دسترسی‌پذیری و سئو.
  • تگ‌های <video> و <audio> برای پخش محتوای چندرسانه‌ای بدون نیاز به افزونه‌ها.
  • کاهش وابستگی به فناوری‌های خارجی مانند فلش.
  • APIهای جدید مانند Canvas، Geolocation، Web Storage، WebSockets و Drag-and-Drop برای ایجاد برنامه‌های وب پیشرفته.
W3C نسخه‌های HTML 5.1 (2016) و HTML 5.2 (2017) را منتشر کرد که بهبودهای جزئی نسبت به HTML5 ارائه می‌دادند. از سال 2019 W3C توسعه نسخه‌های جدید را متوقف کرد و به استاندارد زنده WHATWG پیوست. اکنون HTML به‌عنوان یک استاندارد زنده و بدون نسخه‌بندی ثابت توسط WHATWG توسعه می‌یابد

1.2 اعلان نوع سند

اعلان نوع سند (Document Type Declaration یا DOCTYPE) اولین خط کدی است که در یک سند HTML5 نوشته می‌شود و قبل از تگ <html> قرار می‌گیرد. این اعلان به مرورگر اطلاع می‌دهد که سند موردنظر یک سند HTML است و باید بر اساس استانداردهای مشخصی تفسیر شود. ساختار آن به شکل زیر است: 

<!DOCTYPE html>
این خط ساده و بدون پیچیدگی است و برای HTML5 استفاده می‌شود. برخلاف نسخه‌های قدیمی‌تر HTML (مانند HTML 4.01 یا XHTML) که اعلان‌های پیچیده‌تری داشتند، در HTML5 این اعلان ساده‌سازی شده است.

هدف و اهمیت آن

  • تعیین استاندارد: اعلان DOCTYPE به مرورگر اعلام می‌کند که سند از استاندارد HTML5 پیروی می‌کند. این کار از تفسیر نادرست سند توسط مرورگر جلوگیری می‌کند.
  • جلوگیری از حالت Quirks: اگر DOCTYPE وجود نداشته باشد یا نادرست باشد، مرورگر وارد حالت Quirks Mode می‌شود. در این حالت، مرورگر سند را با قوانین قدیمی و غیراستاندارد رندر می‌کند که می‌تواند منجر به نمایش ناسازگار
  • حالت استاندارد (Standards Mode): با استفاده از <!DOCTYPE html>، مرورگر وارد حالت استاندارد می‌شود و سند را با رعایت استانداردهای مدرن CSS و HTML رندر می‌کند.

تأثیر بر Rendering Mode:

  • Quirks Mode: در این حالت مرورگر رفتارهای قدیمی‌تر و ناسازگار را شبیه‌سازی می‌کند. به عنوان مثال ممکن است باکس‌مدل CSS به‌صورت غیراستاندارد تفسیر شود.
  • Standards Mode: با وجود DOCTYPE صحیح مرورگر از قوانین استاندارد W3C استفاده می‌کند که باعث نمایش یکنواخت و قابل پیش‌بینی صفحه در مرورگرهای مختلف می‌شود.
  • Almost Standards Mode: حالتی میانی که در برخی موارد خاص (مانند استفاده از DOCTYPE‌های قدیمی) فعال می‌شود و رفتارهای خاصی را اعمال می‌کند.
این یک سند HTML5 است.
<!DOCTYPE html>
  <html lang="en">
  <head>
      <title>نمونه سند</title>
  </head>
  <body>
      <p>این یک سند HTML5 است.</p>
  </body>
  </html>
در این مثال، اعلان DOCTYPE تضمین می‌کند که مرورگر سند را به‌صورت استاندارد رندر می‌کند.

عناصر Void, Raw-Text, Escapable-Raw-Text Elements

این مفاهیم به انواع خاصی از عناصر HTML اشاره دارند که نحوه پردازش محتوا و قوانین نوشتاری آن‌ها با سایر عناصر متفاوت است.

  • 1. Void Elements (عناصر تهی):

    این عناصر تگ بسته ندارند و به‌صورت تک‌تگ نوشته می‌شوند. آن‌ها نمی‌توانند محتوای متنی یا عناصر فرزند داشته باشند. مثال‌ها: <br> - <img> - <input> - <meta> - <link> - <hr>. این نوع تگ ها نیازی به تگ بسته (مانند </br>) ندارند. در HTML5 می‌توانند به‌صورت بسته نوشته شوند (مثل <br/>) اما این کار اختیاری است. 

    <img src="image.png" alt="تصویر نمونه">

  • 2. Raw-Text Elements (عناصر متن خام):

    HTML درون آن‌ها به‌عنوان متن ساده نمایش داده می‌شوند و محتوای داخل این عناصر به‌صورت خام تفسیر می‌شود و تگ‌های HTML داخل آن‌ها اجرا نمی‌شوند. مثال‌ها: <script> - <style>. 

    <script>
  alert("Hello World!"); // این کد به‌عنوان جاوااسکریپت اجرا می‌شود
  <p>این تگ اجرا نمی‌شود</p>
</script>

  • 3. Escapable-Raw-Text Elements (عناصر متن خام قابل فرار):

    این عناصر نیز محتوای متنی را می‌پذیرند اما کاراکترهای خاصی (مانند < و &) باید به‌صورت کدهای فرار (escape codes) نوشته شوند تا به‌درستی نمایش داده شوند. مثال‌ها: <textarea> - <title>. کاراکترهای خاص به‌صورت کدهای فرار (مانند &lt; برای < یا &amp; برای &) نوشته می‌شوند.

    این متن به‌ صورت خام نمایش داده می‌شود
    <textarea>
  <p>این متن به‌ صورت خام نمایش داده می‌شود</p>
</textarea>

1.3 تگ‌های ریشه

تگ <html> و ویژگی‌های lang و dir

تگ <html> ریشه اصلی سند HTML است و تمام محتوای سند (به‌جز DOCTYPE) درون آن قرار می‌گیرد. این تگ معمولاً شامل دو ویژگی مهم است:

  • ویژگی lang

    این ویژگی زبان اصلی سند را مشخص می‌کند. این اطلاعات برای دسترسی‌پذیری (accessibility) موتورهای جستجو و ابزارهای ترجمه بسیار مهم است. مقادیر معمول: en (انگلیسی)، fa (فارسی)، ar (عربی)، همچنین میتوانید با مراجعه به این صفحه لیست کامل و دقیقی از آن هارا ببنید. 

    <html lang="fa">
    این کد به مرورگر و ابزارهای دیگر اعلام می‌کند که زبان اصلی سند فارسی است.

  • ویژگی dir

    این ویژگی جهت‌گیری متن (directionality) را مشخص می‌کند. مانند ltr (چپ به راست، مانند انگلیسی)، rtl (راست به چپ، مانند فارسی و عربی). ویژگی dir باعث می‌شود که ساختار صفحه (مانند ترازبندی متن و عناصر) متناسب با جهت‌گیری زبان تنظیم شود. 

    <html lang="fa" dir="rtl">
    این کد مشخص می‌کند که سند به زبان فارسی و با جهت‌گیری راست به چپ است.

تگ‌های <head> و <body>

  • تگ <head>

    این تگ حاوی متاداده‌ها (metadata) و اطلاعاتی است که مستقیماً در صفحه نمایش داده نمی‌شوند اما برای عملکرد و نمایش صحیح صفحه ضروری هستند. مانند: <title>: عنوان صفحه که در نوار عنوان مرورگر یا تب‌ها نمایش داده می‌شود. <meta>: اطلاعاتی مانند کدهاگذاری کاراکترها (مثل <meta charset="UTF-8">)، توضیحات صفحه یا تنظیمات نمایش. <link>: برای اتصال منابع خارجی مانند فایل‌های CSS استفاده می شود. <script>: برای افزودن کدهای جاوااسکریپت. 

    <head>
  <meta charset="UTF-8">
  <title>صفحه نمونه</title>
  <link rel="stylesheet" href="styles.css">
</head>

  • تگ <body>

    این تگ حاوی محتوای اصلی صفحه است که در مرورگر نمایش داده می‌شود مانند متن، تصاویر، لینک‌ها و عناصر تعاملی. مانند: عناصر ساختاری مانند <header> - <nav> - <main> - <footer> و عناصر محتوایی مانند <p> - <img> - <a> - <div>

    خوش آمدید

    این یک پاراگراف نمونه است.
    <body>
  <h2>خوش آمدید</h2>
  <p>این یک پاراگراف نمونه است.</p>
</body>
  • رابطه ی بین <head> و <body>
    • تگ <head> اطلاعات زمینه‌ای و تنظیمات صفحه را فراهم می‌کند، در حالی که <body> محتوای قابل مشاهده را شامل می‌شود. این دو تگ مکمل یکدیگر هستند و هر دو زیرمجموعه تگ <html> هستند. <head> معمولاً قبل از <body> قرار می‌گیرد و مرورگر ابتدا محتوای <head> را پردازش می‌کند تا تنظیمات لازم (مانند کدگذاری یا استایل‌ها) را اعمال کند.

1.4 قواعد نوشتن

Proper Nesting (تودرتو بودن صحیح):

در HTML عناصر باید به‌صورت صحیح و سلسله‌مراتبی تودرتو شوند. این به این معناست که یک تگ بازشده باید قبل از باز شدن تگ دیگر بسته شود مگر اینکه تگ‌ها به‌صورت منطقی اجازه تودرتو شدن داشته باشند. مانند: 

<p>این یک پاراگراف است.</p>
شکل زیر نادرست خواهد بود و در استفاده های پیاپی دچار مشکل می شود و باید تگ حتما بسته شود. 

<p>این یک پاراگراف است.
شکل زیر نیز ناردرست خواهد بود و برای پایان تگ باید علامت اسلش (/) قبل از نام تگ گذاشته شود تا تگ بسته شود. 

<p>این یک پاراگراف است.<p>
تگ‌های بلاک (مانند <div>، <p>) معمولاً می‌توانند تگ‌های دیگر را در خود جای دهند، اما تگ‌های درون‌خطی (مانند <span>، <a>) محدودیت‌هایی دارند. نقض قوانین تودرتو می‌تواند باعث خطاهای رندرینگ یا رفتار غیرمنتظره در مرورگر شود.

Case-Sensitivity (حساسیت به حروف بزرگ و کوچک):

در HTML5 نام تگ‌ها و ویژگی‌ها به حروف بزرگ و کوچک حساس نیستند. یعنی <DIV> - <div> - <DiV> همگی یکسان تفسیر می‌شوند. بهتر است از حروف کوچک استفاده کنید، زیرا این روش استاندارد و خواناتر است. مانند:

این یک div است
این یک div است
<DIV>این یک div است</DIV> <!-- معتبر -->
<div>این یک div است</div> <!-- توصیه شده -->
در برخی موارد خاص (مانند مقادیر ویژگی‌ها) حساسیت به حروف وجود دارد. برای مثال در ویژگی class، مقدار کلاس به حروف بزرگ و کوچک حساس است وقتی با CSS یا جاوااسکریپت کار می‌کنید.

Comments (کامنت‌ها):

کامنت‌ها در HTML با استفاده از <!-- … --> نوشته می‌شوند و برای افزودن توضیحات یا غیرفعال کردن بخشی از کد استفاده می‌شوند. محتوای داخل کامنت‌ها توسط مرورگر نادیده گرفته می‌شود و در صفحه نمایش داده نمی‌شود. از کامنت‌ها برای توضیح کد، عیب‌یابی یا مستندسازی استفاده می‌شود.

این متن نمایش داده می‌شود.
<!-- این یک کامنت است -->
<p>این متن نمایش داده می‌شود.</p>
<!-- 
این یک کامنت چندخطی است
برای توضیحات بیشتر
-->
کامنت‌ها نباید تودرتو شوند (یعنی قرار دادن <!-- --> داخل یک کامنت دیگر). از کامنت‌ها برای غیرفعال کردن موقت کد نیز می‌توان استفاده کرد: 

<!-- <p>این پاراگراف نمایش داده نمی‌شود</p> -->
